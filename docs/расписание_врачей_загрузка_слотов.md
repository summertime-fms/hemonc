# Загрузка расписания и слотов врачей (онлайн-запись)

Документ для агента, настраивающего **клон сайта** (1С-Битрикс). Описывает полный путь от клика «Записаться» до отображения календаря и слотов, внешние зависимости и типичные причины, когда вместо расписания показывается форма «Для записи позвоните нам».

---

## 1. Краткая схема

```
[Клик «Записаться»]
        │
        ▼
JS: ShowPersonalDoctorPopup(bitrixElementId)     ← это ID элемента инфоблока, НЕ GUID 1С
        │
        ▼
POST /ajax/  { action: 'GetDoctor', doctorId: <bitrix_id> }
        │
        ▼
PHP: site/ajax/index.php
  → ElementDoctorsTable::getByPrimary(doctorId)
  → читает свойство ONES_GUID врача
  → Hemonc\Ajax::getDoctorSchedule(ONES_GUID)  → HTTP к 1С
        │
        ├─ расписание есть → HTML календаря + initDoctorDialog(ONES_GUID, CLINIC_GUID)
        │
        └─ расписания нет  → HTML «Для записи позвоните нам» (fallback)
                │
                ▼
[Выбор даты]
        │
        ▼
POST /ajax/  { action: 'GetHours', doctorID: <ONES_GUID>, date: <YYYYMMDD> }
        │
        ▼
Hemonc\Ajax::getDoctorHours() → HTTP к 1С → HTML со слотами времени
        │
        ▼
[Отправка записи]
POST /ajax/  { action: 'SetOrder', doctorid, clinicId, datetime, patient }
        │
        ▼
Hemonc\Ajax::setOnDoctorHour() → HTTP к 1С
```

---

## 2. Ключевое: два разных идентификатора врача

| Идентификатор | Где используется | Что это |
|---------------|------------------|---------|
| **Bitrix ID** (`doctorId`) | Фронтенд: `ShowPersonalDoctorPopup(123)`, `data-id="123"` | `b_iblock_element.ID` инфоблока «Врачи» (IBLOCK_ID = **15**) |
| **ONES_GUID** | Запросы к 1С, `GetHours`, `SetOrder` | UUID врача в 1С (свойство `ONES_GUID` элемента) |

**На клоне другие Bitrix ID — это нормально.** Главное, чтобы у каждого врача в админке был заполнен корректный `ONES_GUID` из 1С клона/клиники.

Ответ с текстом «Для записи позвоните нам или оставьте номер» (как в примере для Шорниковой) означает: врач в Битрикс **найден** (имя и фото подставляются), но сервер решил, что **онлайн-расписание недоступно** (`$notAvail = true` в `site/ajax/index.php`).

## 4. AJAX API (`POST /ajax/`)

Все запросы — **только POST**. GET возвращает `False`.

### 4.1. `action=DoctorsSelectPopup`

Список врачей для модалки «Выберите врача».

**Параметры:** только `action`.

**Логика:**
1. Все активные врачи из IBLOCK 15.
2. `Hemonc\Ajax::getSchedule()` — ближайшее время по всем врачам из 1С.
3. Если для `ONES_GUID` врача есть ближайший слот — карточка кликабельна (`ShowPersonalDoctorPopup(ID)`).

### 4.2. `action=GetDoctor` — загрузка календаря

**Параметры:**

| Параметр | Обязательный | Описание |
|----------|--------------|----------|
| `doctorId` | да | Bitrix ID элемента врача |
| `inline` | нет | `true` — встроенный вид (страница услуги), иначе модалка |

**Условия показа fallback («позвоните нам»):**

```php
// site/ajax/index.php, action GetDoctor
$notAvail = true если:
  - не передан doctorId
  - врач не найден / неактивен
  - пустой ONES_GUID
  - getDoctorSchedule(ONES_GUID) вернул пустой массив
  - проверка дат в цикле (см. §7 — известная особенность)
```

**Успешный ответ:** HTML с календарём на 2 месяца, блоком времени, формой телефона и скриптом:

```javascript
initDoctorDialog("<ONES_GUID>", "<CLINIC_GUID>", true|false);
```

### 4.3. `action=GetHours` — слоты на выбранную дату

**Параметры:**

| Параметр | Обязательный | Описание |
|----------|--------------|----------|
| `doctorID` | да | **ONES_GUID** врача (не Bitrix ID!) |
| `date` | да | Дата в формате `YYYYMMDD` (из `data-date` ячейки календаря) |

**Ответ:** HTML-блок `.dnc-time-select` со слотами `HH:00` или заглушками `..:..`.

### 4.4. `action=SetOrder` — подтверждение записи

**Параметры:**

| Параметр | Описание |
|----------|----------|
| `doctorid` | ONES_GUID врача |
| `clinicId` | CLINIC_GUID клиники |
| `datetime` | `YYYYMMDD` + `HHMM` (например `202607021400`) |
| `patient` | Телефон пациента |

**Ответ:** `Ok` или `false`.

После успеха JS вызывает CallTouch и цель Яндекс.Метрики.

## 5. Интеграция с 1С (OncoHTTPUpdate)

Класс: `Hemonc\Ajax` в `site/local/php_interface/AjaxClass.php`.

**Базовый URL:**

```
https://laskov.medconsult.ru/laskov/hs/OncoHTTPUpdate/ScheduleUpdate?action=<method>&<params>
```

**Авторизация:** HTTP Basic Auth  
- логин: константа `ONESUSER` (`wsuser` в `init.php`)  
- пароль: константа `ONESPWD` (`456qwe` в `init.php`)

**Клиники:** массив `CLINICS_GUID` в `init.php`:

```php
define('CLINICS_GUID', [
    'e75d8aab-9952-11ec-80fe-002590c014a5', // Болотниковская
]);
```

Запросы расписания идут **по каждому GUID из этого массива**.

### 5.1. Методы 1С

| Метод `action` | PHP-метод | Параметры | Назначение |
|----------------|-----------|-----------|------------|
| `nearestTimeWeekAll` | `getSchedule()` | `Klinika` | Ближайшее время для всех врачей; ответ: массив строк `"время_GUIDврача"` |
| `getMonth` | `getDoctorSchedule($doctorGUID)` | `doctorid`, `month` (`Ym`), `Klinika` | Даты приёма на месяц; ответ: массив дат `dd.mm.yyyy` |
| `get` | `getDoctorHours($doctorGUID, $date)` | `doctorid`, `date` (`YYYYMMDD`), `Klinika` | Свободные часы; ответ: массив `"HH:00"` |
| `set` | `setOnDoctorHour(...)` | `doctorid`, `Klinika`, `patient`, `datetime` | Создание записи |

### 5.2. Кэширование

| Данные | TTL | Ключ кэша |
|--------|-----|-----------|
| Ближайшее время всех врачей | 70 сек | `Hemonc_Ajax_getSсhedule` |
| Месяцы врача | 75 сек | `Hemonc_Ajax_getDoctorSchedule_{ONES_GUID}` |
| Часы на дату | без кэша | — |

Кэш-директория Битрикс: `Hemonc_Ajax`. После смены GUID или правок в 1С — **очистить кэш** (`/bitrix/admin/cache.php` или `php -f clear_cache.php`).

## 6. Форматы дат (важно для отладки)

| Этап | Формат | Пример |
|------|--------|--------|
| Ключи в `$doctorSchedule` от 1С | `dd.mm.yyyy` | `15.07.2026` |
| Атрибут `data-date` в календаре | `YYYYMMDD` | `20260715` |
| Атрибут `data-dmy` в календаре | `dd.mm.yyyy` | `15.07.2026` |
| Запрос `GetHours` | `YYYYMMDD` | `20260715` |
| Запрос `SetOrder` datetime | `YYYYMMDD` + `HHMM` | `202607151400` |

Сопоставление доступных дней в календаре: `array_key_exists($day['dmy'], $doctorSchedule)`.

## 7. Известная особенность в коде проверки дат

В `site/ajax/index.php` (action `GetDoctor`) после получения `$doctorSchedule` выполняется цикл:

```php
foreach ($doctorSchedule as $doctorDate => $guid) {
    if (strtotime($doctorDate) > strtotime('now')) {
        $notAvail = false;
    } else {
        $notAvail = true;
    }
}
```

Итоговое значение `$notAvail` зависит от **последней** даты в массиве. Если 1С вернула только прошедшие даты — покажется fallback, даже при наличии `ONES_GUID`.

## 8. Чеклист для клона сайта (почему не грузится расписание)

### 8.1. Данные врачей в Битрикс

- [ ] У врача заполнено свойство **`ONES_GUID`** (UUID из 1С клона, не с оригинального сайта).
- [ ] У врача заполнено свойство **`CLINIC_GUID`** (UUID клиники в 1С).
- [ ] Врач **активен** (`ACTIVE = Y`).
- [ ] `doctorId` в AJAX — это **ID из БД клона**, не с оригинала (проверить в DevTools → Network → POST `/ajax/`).

**Проверка в админке:**  
`Контент → Врачи (iblock 15) → элемент → свойства ONES_GUID / CLINIC_GUID`.

**Проверка в БД:**

```sql
SELECT e.ID, e.NAME, p1.VALUE AS ONES_GUID, p2.VALUE AS CLINIC_GUID
FROM b_iblock_element e
LEFT JOIN b_iblock_element_property p1 ON p1.IBLOCK_ELEMENT_ID = e.ID AND p1.IBLOCK_PROPERTY_ID = 34
LEFT JOIN b_iblock_element_property p2 ON p2.IBLOCK_ELEMENT_ID = e.ID AND p2.IBLOCK_PROPERTY_ID = 35
WHERE e.IBLOCK_ID = 15 AND e.ACTIVE = 'Y' AND e.NAME LIKE '%Шорникова%';
```

(ID свойств 34/35 — из дампа оригинала; на клоне сверить `b_iblock_property.CODE`).

### 8.2. Конфигурация сервера

- [ ] В `site/local/php_interface/init.php` заданы `ONESUSER`, `ONESPWD`.
- [ ] Массив **`CLINICS_GUID`** содержит GUID клиник, для которых в 1С настроено расписание клона.
- [ ] С сервера клона доступен `https://laskov.medconsult.ru/...` (firewall egress, DNS, HTTPS).
- [ ] IP сервера клона добавлен в белый список на стороне 1С (если используется).

### 8.3. Ручная проверка API 1С

С сервера клона (curl):

```bash
curl -u 'wsuser:456qwe' \
  'https://laskov.medconsult.ru/laskov/hs/OncoHTTPUpdate/ScheduleUpdate?action=getMonth&doctorid=<ONES_GUID>&month=202607&Klinika=<CLINIC_GUID>'
```

Ожидается JSON-массив дат `["01.07.2026","05.07.2026",...]`. Пустой массив `[]` или ошибка — причина fallback на сайте.

Для слотов на дату:

```bash
curl -u 'wsuser:456qwe' \
  'https://laskov.medconsult.ru/laskov/hs/OncoHTTPUpdate/ScheduleUpdate?action=get&doctorid=<ONES_GUID>&date=20260715&Klinika=<CLINIC_GUID>'
```

### 8.4. Кэш и код

- [ ] Очищен кэш Битрикс после импорта врачей / смены GUID.
- [ ] Файлы `site/ajax/index.php` и `site/local/php_interface/AjaxClass.php` перенесены с оригинала (не устаревшая версия).
- [ ] Подключены JS: `doctorsPopup.js`, `calendar.js` (см. `header.php` шаблона).

### 8.5. Типичная ошибка при клонировании

| Симптом | Вероятная причина |
|---------|-------------------|
| Fallback с именем и фото врача | Врач есть в БД, но `ONES_GUID` пустой, неверный, или 1С не отдаёт расписание для этого GUID |
| Кнопка «Записаться» ведёт на fallback, на оригинале работает | У клона другой `ONES_GUID` / не синхронизирован с 1С клона |
| Календарь есть, слоты `..:..` | `GetHours` не получает часы: неверный `doctorID` (должен быть ONES_GUID), неверный `date`, нет слотов в 1С |
| Запись не создаётся | Неверный `CLINIC_GUID`, недоступен `action=set`, ошибка авторизации 1С |

## 9. Отладка через браузер

1. Открыть DevTools → Network.
2. Нажать «Записаться» → найти `POST /ajax/` с `action=GetDoctor&doctorId=...`.
3. Если в ответе HTML с `modal-title">Для записи позвоните` — проблема на бэкенде (§8), не во фронтенде.
4. Если в ответе календарь — выбрать дату → `POST /ajax/` с `action=GetHours` → проверить `doctorID` (должен быть UUID, не числовой Bitrix ID).
5. Временно добавить лог в `AjaxClass::sendOnesRequest()` или смотреть `/local/debug/` (используется для CallTouch).

## 10. Связанные интеграции (не 1С, но в том же flow)

| Система | Где | Когда |
|---------|-----|-------|
| CallTouch | `calendar.js` → `SendCalltouchRequest()` | После успешной `SetOrder` |
| Bitrix24 форма | `site/local/templates/hemonc_h/parts/bitrixCallback.php` | В fallback «позвоните нам» |
| Яндекс.Метрика | `doctorsPopup.js`, `calendar.js` | Цели `doctor-form-{id}`, `Запись на прием` |

На клоне ID форм Bitrix24 и счётчиков могут отличаться — это влияет только на fallback и аналитику, **не на загрузку слотов**.

## 11. Минимальный сценарий приёмки на клоне

1. В админке у тестового врача заполнены `ONES_GUID` и `CLINIC_GUID` из 1С.
2. `curl getMonth` с сервера возвращает непустой список дат.
3. `POST /ajax/` `GetDoctor` + `doctorId` → HTML с «Выберите дату и время» и календарём.
4. Клик по доступной дате → `GetHours` → слоты с реальным временем.
5. Отправка тестовой записи → ответ `Ok` (или согласованная обработка на стороне 1С).
